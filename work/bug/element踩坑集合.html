    一. 关于scoped的坑

    鹏 の 博客

    # 一. 关于scoped的坑

    • 当我使用一些ui组件的时候，某些样式我们可能需要修改以达到设计图的要求，此时就不得不修改其本身的样式了，但是我奇怪的发现样式加不上，面向百度编程后发现原来是scoped在作怪，我们先来看下scoped在vue中的作用

    实现组件的私有化，不对全局造成样式污染，表示当前style属性只属于当前模块

    • 我们打开控制台可以看到每一个标签上加上了唯一标识，这就是<style scoped>加上后的效果
    • 当我们把scoped去掉后发现可以修改组件的属性了
      总结：当我们需要修改第三方ui库的样式的时候切记把scoped去掉，原因在于第三方ui库一般在全局就进行引入了，当加上scoped这个实现组件私有化属性后父级就无法设置子组件的属性了，当去掉scoped后我们书写类名的时候就需要我们规范点了，避免和其他组件发生冲突

    # 二. 关于form表单相关

    • 在el-form组件中有一个required属性，代表是必填项会出现*号标注，但是当校验的时候规则里会填这个字段{ required: true, message: '请输入活动名称', trigger: 'blur' }，当在el-form-item和校验规则都填写required这个属性的话，校验规则提示会不知觉触发英文校验，解决办法就是把el-form-item上的required的这个属性去掉就ok了
    • 在校验的时候每个el-form-item上都需要加上prop属性，但是必须注意的是prop所绑定的值需要和v-model所绑定的值相对应才能够对表单正常地进行校验
    • 当需要进行表单数据回显的时候，我选择的做法是在created中拿到数据对v-model绑定的值进行赋值，此时需要注意的是我们绑定的是value值而不是lable值，当进行联级选择器的地区回显时，我发现不能正常地进行回显，加上change事件发现打印的值是字符串的数组，而后台给的数据时字符字符串，因此我们需要在回显的时候对数据进行处理一下，用split方法以,分割，然后发现数据可以正常地进行回显了

    # 三.关于table表单相关

    • 当我们需要在table表单中嵌套按钮或者其他组件的时候，我们需要在对应地方加上<template slot-scope="scope"></template>然后在里面写上对应的组件，如果我们需要拿到当行对应的数据去做对应的操作的话，我们可以可以在el-table上加上row-click事件
